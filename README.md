# Dev Playbook

Indice detalhado: `INDEX.md`

Um repositório pessoal onde registro padrões, decisões e aprendizados relacionados ao desenvolvimento de software.

A ideia é manter um **manual vivo** do meu processo de desenvolvimento:
como estruturo projetos, quais boas práticas sigo e quais lições aprendi ao longo do caminho.

## Objetivos

* Organizar conhecimento técnico adquirido durante meus estudos e projetos.
* Registrar decisões de desenvolvimento e aprendizados importantes.
* Criar um guia de referência para futuros projetos.

---

## Estrutura do repositório

### Playbook

Guias práticos sobre **como eu construo meus projetos**.

Exemplos:

* Estrutura inicial de projetos
* Organização de arquivos
* Padrões de commits
* Checklist antes de publicar um projeto

Esses documentos funcionam como um **manual de execução** para iniciar e manter projetos.

---

### Conhecimento

Notas técnicas sobre desenvolvimento front-end e boas práticas.

Exemplos:

* Acessibilidade em interfaces
* Boas práticas de HTML
* Organização de CSS
* Padrões de JavaScript
* Responsividade

Aqui ficam **conceitos, padrões e observações técnicas** que considero importantes lembrar.

---

### Componentes

Pequenos padrões de interface reutilizáveis.

Exemplos:

* Menu hambúrguer acessível
* Header responsivo
* Cards acessíveis
* Layouts reutilizáveis

Cada componente documenta:

* o problema que resolve
* a solução adotada
* o motivo da decisão

---

### Aprendizados

Registros curtos sobre decisões e lições tiradas de projetos reais.

Exemplo de registro:

Data — Contexto do projeto
Problema encontrado
Decisão tomada
Aprendizado obtido

Essa seção funciona como um **registro da minha evolução como desenvolvedora**.

---

## Como uso este repositório

Sempre que aprendo algo relevante durante:

* estudos
* desenvolvimento de projetos
* leitura de documentação
* resolução de problemas

registro aqui de forma simples e objetiva.

A ideia não é escrever tutoriais completos, mas **notas técnicas claras que eu possa consultar no futuro**.

---

## Filosofia

Software não é apenas código que funciona.
Ele precisa gerar valor e também ser **compreensível, utilizável e sustentável ao longo do tempo**.

Este repositório existe para me ajudar a evoluir nessa direção.
